Guía para la Api Rest

Base url: '/api'

GETs

/app_authorization

Descripción

Endpoint que sirve para autenticar el uso del App.

Params:
  • clientId: xxx
  • clientSecret: xxx
Return 200:
{ "error": false, "message": "Success", "code": "VALID", "access_token": "xxx" }
Return 400:
{ "error": true, "message": "security.invalidCredentials", "code": "INVALID_TOKEN", "access_token": "" }

/authentication_user

Descripción

Endpoint que sirve para realizar el login de usuario.

Params:
  • userName: xxx
  • password: xxx
  • header: {Authorization : Bearer xxxx}
Return 200:
{ "error": false, "message": "Success", "code": "VALID", "access_token": "xxx" }
Return 400:
{ "error": true, "message": "security.invalidCredentials", "code": "INVALID_TOKEN", "access_token": "" }

/get_user_info

Descripción

Obtiene la información del usuario logeado mediante el Token.

Params:
  • header: {Authorization : Bearer xxxx}
Return 200:
{ "id": 000, "lastName": "xxx", "name": "Test", "country": { "id": 0 }, "error": false, "message": "" }
Return 400:
{ "error": true, "message": "INVALID_TOKEN", "id": 0, "lastName": "", "name": "", "country": { "id": 0 } }

/search_restaurants

Descripción

Pasando coordenasdas devuelve los restaurantes más cercanos.

Params:
point : req.query.point, country : req.query.country, max : req.query.max, offset: req.query.offset, sort : req.query.sort, count : req.query.count, fields : req.query.fields
  • point: xxx,xxx required!
  • country: xxx required!
  • max: xxx
  • offset: xxx
  • sort: xxx
  • count: xxx
  • fields: xxx
Return 200:
{ "error": false, "message": "", "restaurants": [...] }
Return 400:
{ "error": true, "message": "", "restaurants": [] }
